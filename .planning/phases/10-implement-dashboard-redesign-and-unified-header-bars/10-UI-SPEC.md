## Typography

Four sizes, two weights:

| Role | Size | Weight | Line Height | Used For |
|------|------|--------|-------------|----------|
| Display | 18px (`text-lg`) | 600 (`font-semibold`) | 1.2 | Dashboard greeting "Good morning, Luis" |
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Task titles, section content, descriptions |
| Label | 11px (`text-[11px]`) | 400 (`font-normal`) | 1.3 | Sync cluster text ("Synced", "3m ago"), blocked reason (italic variant), nudge body text |
| Micro | 9px (`text-[9px]`) | 400 (`font-normal`) | 1.0 | Badge text (h-4 badges), personal badge, pending count |

**Weight exceptions:**

| Context | Weight |
|---------|--------|
| Section headings ("Today's Focus", "Upcoming", "Blocked") | 600 (`font-semibold`) at 14px |
| Page name in top bar left zone | 600 (`font-semibold`) at 14px |
| Stats summary line ("3 in progress · 1 blocked") | 400 at 14px, muted-foreground color |
| Nudge count emphasis (`<strong>`) | 600 (`font-semibold`) at 14px (`text-sm`) — mapped to Body size |
| Greeting sub-line | 400 at 14px, muted-foreground |
| Due date (amber or normal) | 400 at 11px (`text-[11px]`) — maps to Label size; amber variant uses `text-amber-600`; tabular-nums |

Heading line-height: 1.2. Body line-height: 1.5. Label line-height: 1.3.

## Layout Contract

### AppShell Top Bar — 3-Zone Pattern

```
┌─────────────────────────────────────────────────────────────────┐
│ [Page name] [divider] [SyncCluster]  │  [#topbar-center]  │  [Search ⌘K] [+ New ▾] [🕐] │
│ ← LEFT ZONE (shrink-0)              │  ↑ flex-1 center   │  ← RIGHT ZONE (shrink-0)    │
└─────────────────────────────────────────────────────────────────┘
  h-[46px], titlebar-drag outer, titlebar-no-drag on all interactive children
```

**Left zone:**
- Page name: `text-sm font-semibold text-foreground select-none shrink-0 titlebar-no-drag`
- Divider: `w-px h-4 bg-border mx-1 shrink-0`
- SyncCluster component: `titlebar-no-drag shrink-0`

**Center zone:**
- `id="topbar-center"`, `class="flex-1 flex items-center justify-center gap-2 titlebar-no-drag"`
- Each view teleports its content here via `<Teleport v-if="isActive" to="#topbar-center">`

**Right zone:** Unchanged from current production.

### Dashboard 2-Column Layout

```
┌──────────────────────────────────────────────────────┐
│  Greeting + Summary                                  │  (full width)
│  Attention Bar (horizontal scroll, conditional)      │  (full width)
│  Stats line                                          │  (full width)
├────────────────────────────────┬─────────────────────┤
│  LEFT COLUMN (w-3/5)           │  RIGHT COLUMN (w-2/5)│
│  ─ Today's Focus               │  ─ Needs Your Review │
│  ─ Upcoming                    │  ─ Your Pull Requests│
│  ─ Blocked (conditional)       │  ─ Pipelines         │
└────────────────────────────────┴─────────────────────┘
```

### Dashboard Task Row Anatomy (Today's Focus + Upcoming)

```
[●priority-dot] [icon/SquareCheckBig] [task title truncate] [personal badge?] [sync dot?] [status badge] [due date]
 size-2           14px icon             flex-1 text-sm        text-[8px]         size-1.5    text-[9px] h-4  text-[11px]
```
Row padding: `px-4 py-2`. Hover: `hover:bg-muted/50 transition-colors`. Cursor: `cursor-pointer`.

### Blocked Section Row Anatomy

```
[●priority-dot] [task title truncate]
                [blocked reason text — italic, red-tinted, pl-4]
```
Section container: `rounded-lg overflow-hidden border border-border border-l-2 border-l-red-500`

## Interaction Contract

### Attention Bar

| Interaction | Behavior |
|-------------|---------|
| Horizontal overflow | `overflow-x-auto` CSS — no JS scroll logic |
| Nudge visibility | `v-if` conditional per nudge type — shown only when data exists. No animation required (enters/exits on data change). |
| Nudge click | No action required in this phase — nudges are informational only |
| Empty Attention Bar | Entire bar hidden (`v-if="showAttentionBar"` where `showAttentionBar = dueSoonTasks.length > 0 || failedPipelines.length > 0 || mergeReadyPRs.length > 0`) |

### Sync Cluster

| Interaction | Behavior |
|-------------|---------|
| Refresh button click | Calls `syncStore.manualSync()`. Button must carry `aria-label="Refresh sync"` (icon-only — no visible text label). |
| While syncing | Refresh button disabled, icon swaps to `Loader2` with `animate-spin` |
| Connected state | Driven by `adoStore.connected` (green dot / red dot) |

### Dashboard Task Rows

| Interaction | Behavior |
|-------------|---------|
| Row hover | `hover:bg-muted/50 transition-colors` (150ms ease, from global style) |
| Row click | Opens TaskDetail panel (existing router/store logic — no change) |
| Priority dot | Display only, no click action |
| Personal badge | Display only |
| Pending sync dot | Display only (omit if `task.pendingSync` field unavailable — do not block shipping on this) |

### Header Center Zone Teleport

| Interaction | Behavior |
|-------------|---------|
| Page navigation | `v-if="isActive"` guard on each `<Teleport>` — ensures only active page's content renders in `#topbar-center` |
| `isActive` implementation | Use `useRoute().name === 'dashboard'` (or equivalent route name) pattern established in existing views |
| All interactive elements in center zone | Must have `titlebar-no-drag` class (already on `#topbar-center` div) |

## Field Existence Decisions

These fields appear in the playground but have uncertain production status. The executor MUST resolve before implementing:

| Field | Uncertainty | Decision |
|-------|-------------|---------|
| `task.pendingSync` | Not on production `Task` type | **Omit amber sync dot** from DashboardTaskRow. Do not add backend field in this phase. |
| `task.blockedReason` | Check `domain/task.go` | If field exists: show italic red-tinted text below title. If missing: show title only in blocked row (no reason text). Do not add field in this phase if absent. |
| `task.isPersonal` | Not a stored field — derived | Compute via `taskStore.isPublicTask(task.id)` returning boolean; `isPersonal = !isPublic`. |
| `syncStore.pendingCount` | `pendingDiff` is single diff, not count | Use `syncStore.conflicts.length` for pending badge count. Hide badge when 0. |

## Anti-Patterns (Do Not Do)

| Anti-Pattern | Correct Approach |
|-------------|-----------------|
| Rename `#topbar-actions` in AppShell without updating all 3 existing teleport views | Update AppShell + DashboardView + TasksView + AdoView in the same task |
| Duplicate sync cluster markup in each view's teleport | Extract to `SyncCluster.vue`; render once in AppShell left zone |
| Put `SyncCluster` inside a view's teleport block | SyncCluster lives in AppShell template — not teleported from views |
| Use `TaskRow.vue` for dashboard richer rows | Create `DashboardTaskRow.vue` — different layout, different props |
| Reference `task.pendingSync` without confirming field exists | Omit pending sync dot per decision above |
| Place Attention Bar after stats line | Order: Greeting → Attention Bar → Stats line → Dashboard sections |
| Add `titlebar-drag` to interactive elements in center zone | Center div already has `titlebar-no-drag`; do not override |
