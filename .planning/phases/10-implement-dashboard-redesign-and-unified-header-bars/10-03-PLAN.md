<task type="auto">
  <name>Task 2: Fix dueSoonTasks date comparison to use start-of-day normalization</name>
  <files>frontend/src/views/DashboardView.vue</files>
  <action>
**Gap 2 root cause:** `dueSoonTasks` computed (lines 86-92) uses `(new Date(t.dueDate).getTime() - Date.now()) / (1000 * 60 * 60 * 24)` which does timestamp-level comparison. `new Date('YYYY-MM-DD')` creates midnight UTC, so once local time passes midnight UTC, today's tasks get negative diffDays and are excluded. The existing `startOfDay()` helper in `stores/tasks.ts` already handles this correctly but wasn't used.

**Fix DashboardView.vue:**

1. Add a local `startOfDay` helper at the top of the `<script setup>` (after imports, before computeds). Do NOT import from stores/tasks.ts since it's not exported — just replicate the 3-line helper:
   ```typescript
   function startOfDay(d: Date): Date {
     return new Date(d.getFullYear(), d.getMonth(), d.getDate())
   }
   ```

2. Replace the `dueSoonTasks` computed (lines 86-92). Current:
   ```typescript
   const dueSoonTasks = computed(() =>
     taskStore.tasks.filter(t => {
       if (!t.dueDate || t.status === 'done' || t.status === 'cancelled') return false
       const diffDays = (new Date(t.dueDate).getTime() - Date.now()) / (1000 * 60 * 60 * 24)
       return diffDays >= 0 && diffDays <= 3
     })
   )
   ```
   Replace with:
   ```typescript
   const dueSoonTasks = computed(() =>
     taskStore.tasks.filter(t => {
       if (!t.dueDate || t.status === 'done' || t.status === 'cancelled') return false
       const dueDay = startOfDay(new Date(t.dueDate))
       const today = startOfDay(new Date())
       const diffDays = (dueDay.getTime() - today.getTime()) / (1000 * 60 * 60 * 24)
       return diffDays >= 0 && diffDays <= 3
     })
   )
   ```
   This normalizes BOTH the due date and current date to midnight local time before computing the difference, so today's tasks always have diffDays=0 and are correctly included.
