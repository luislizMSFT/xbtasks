
**2. Replace all `<TaskRow>` usages with `<TreeTaskRow>`:**

For every occurrence of `<TaskRow :task="..." ...>` in the template, replace with `<TreeTaskRow>` and map props:

Old pattern:
```html
<TaskRow
  :task="task"
  :is-public="taskStore.isPublic(task.id)"
  :selected="taskStore.selectedTaskId === task.id"
  @select="selectTask"
  @toggle-status="..."
/>
```

New pattern:
```html
<TreeTaskRow
  :task="task"
  :is-public="taskStore.isPublic(task.id)"
  :selected="taskStore.selectedTaskId === task.id"
  :ado-meta="adoMeta.getAdoMeta(task.id)"
  @click="selectTask(task.id)"
  @toggle-status="(id) => { const t = taskStore.tasks.find(x => x.id === id); if (t) toggleDone(t) }"
/>
```

Key prop/emit changes:
- `@select="selectTask"` → `@click="selectTask(task.id)"` (TreeTaskRow emits `click` without args, not `select` with id)
- Add `:ado-meta="adoMeta.getAdoMeta(task.id)"` prop
- `@toggle-status` stays the same pattern but TreeTaskRow emits `toggleStatus` with task id

Search for ALL `<TaskRow` in the file and replace each one. The file may have multiple contexts (Today's Focus section, Upcoming section, etc.).

**3. Add `toggleDone` function if not already present:**

The DashboardView may not have a toggleDone function — check if it exists. If task status toggling is not currently wired, add:
```typescript
async function toggleDone(task: Task) {
  const newStatus = task.status === 'done' ? 'todo' : 'done'
  await taskStore.setStatus(task.id, newStatus)
}
```

Also add `selectTask` function if missing:
```typescript
function selectTask(id: number) {
  router.push({ name: 'tasks', query: { taskId: String(id) } })
}
```
