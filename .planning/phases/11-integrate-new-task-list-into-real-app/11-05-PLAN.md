Fix 6 specific issues in TasksView.vue:

**Gap A — Drag handle invisible (line 517):**
Change the drag handle class from `text-muted-foreground/0 hover:text-muted-foreground/30` to `text-muted-foreground/40 hover:text-muted-foreground/70`. The `/0` makes it fully transparent and users can't see or find the drag grip dots.

**Gap B — 'No Project' filter hides all group headers (line 366):**
Remove the `.filter(k => k !== 'No Project')` from the grouped rendering `v-for`. Currently line 366 reads:
```html
<div v-for="key in groupKeys.filter(k => k !== 'No Project')" :key="key">
```
Change to:
```html
<div v-for="key in groupKeys" :key="key">
```
This ensures ALL group keys render as headers, including 'No Project'. The existing `groupKeys` computed already sorts 'No Project' last, which is correct.

**Gap C — Group headers not adaptive for status/priority (lines 364-426):**
The current group header template is project-centric: it shows ADO type icons, project badges, and handles `selectProject(key)`. When `groupBy` is 'status' or 'priority', this is wrong — there is no project to select or show icons for.

Refactor the group header to adapt based on `taskStore.groupBy`:

1. For `groupBy === 'project'`: Keep current behavior (ADO type icon, project badge, click-to-select-project).
2. For `groupBy === 'status'`: Show a colored status dot (`statusBgColor(key)`) before the group label. No project-click behavior. Import `statusBgColor` from `@/lib/styles`.
3. For `groupBy === 'priority'`: Show a priority dot (`priorityDotBgColor(key)`) before the group label. No project-click behavior. Import `priorityDotBgColor` from `@/lib/styles`.

The header layout stays the same (chevron + icon/dot + label + count), just the icon/dot and click behavior change. The `groupLabel(key)` function already handles label text for all modes.

Remove `@click.self="selectProject(key)"` from the outer div — replace with: only call `selectProject(key)` when `taskStore.groupBy === 'project'`.
