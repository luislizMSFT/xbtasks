# Phase 13 — UI Design Contract

> Visual and interaction contract for Phase 13: Wikis, Bookmarks & ADO Wiki Integration.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Design language:** Professional, information-dense, clean — Obsidian/Notion hybrid aesthetic.

## Spacing Scale

Inherited from Phase 01. No changes.

| Token | Value | Usage in Phase 13 |
|-------|-------|-------------------|
| xs | 4px | Icon gaps in sidebar page list items, wiki link inline padding |
| sm | 8px | Wiki sidebar item padding, search result gaps, bookmark chip gaps |
| md | 16px | Wiki content area padding, editor/preview internal padding, card padding |
| lg | 24px | Section gaps within WikiView (sidebar header → list → footer), content heading spacing |
| xl | 32px | Wiki content area horizontal padding (left + right) |
| 2xl | 48px | Not used in Phase 13 |
| 3xl | 64px | Not used in Phase 13 |

Exceptions:
- **Wiki sidebar width:** 280px fixed. Not on 8-point scale — layout constant matching Obsidian/VS Code pattern. Source: RESEARCH.md recommendation.
- **Markdown preview line-height:** 1.6 (26px at 16px body) for readability of prose content. Slightly looser than standard 1.5 for sustained reading.
- **Search result snippet padding:** 12px horizontal. Same exception as Phase 01 button padding — midpoint between 8px and 16px for compact interactive elements.

## Interaction States

### Wiki Page CRUD

| Action | Trigger | Behavior |
|--------|---------|----------|
| Create page | Click "New Page" button in sidebar | Creates page with title "Untitled", selects it, enters edit mode, focuses title for rename |
| Select page | Click page in sidebar list | Loads page content in content area. Preserves edit/preview mode |
| Rename page | Double-click title in WikiPageHeader | Title becomes editable input. Enter or blur saves. Escape cancels |
| Edit content | Switch to "Edit" tab | Shows textarea with raw markdown. Auto-save on blur or after 2s debounce of typing inactivity |
| Preview content | Switch to "Preview" tab | Renders markdown → HTML via `useMarkdown`. Wiki links clickable |
| Delete page | Click trash icon in WikiPageHeader | Confirmation dialog: "Delete '{page title}'? This cannot be undone." Confirm = destructive button |
| Wiki link click | Click `[[Page Name]]` in preview | If page exists → select it. If page doesn't exist → create it with that title and select it |

### Search

| Action | Trigger | Behavior |
|--------|---------|----------|
| Type in search | Keystroke in sidebar search input | Debounce 200ms, then FTS5 query. Results replace page list in sidebar |
| Search results | FTS5 response | Show page title + snippet preview (with `<mark>` highlights). Click result → select page |
| Clear search | Clear input or press Escape | Restore full page list |
| No results | FTS5 returns empty | Show inline text "No pages matching '{query}'" below search input |

### ADO Wiki Import

| Action | Trigger | Behavior |
|--------|---------|----------|
| Open browser | Click "Browse ADO Wikis" in sidebar | Opens Dialog. Fetches wiki list from all configured orgs |
| Select wiki | Choose from dropdown | Fetches page tree for selected wiki. Shows loading spinner during fetch |
| Select pages | Toggle checkboxes | Count updates in bottom bar. Parent checkbox toggles all children |
| Import | Click "Import Selected" | Dialog closes. Toast: "Importing {N} pages..." Backend fetches content in background goroutine. Events: `wiki:import:progress`, `wiki:import:complete` |
| Import complete | Backend event | Toast: "Imported {N} pages from {wiki name}". Pages appear in sidebar with ADO badge |
| Re-fetch | Background timer (1 hour default) | Silently updates content of imported pages. No UI indication unless content changed |

### Project Bookmarks

| Action | Trigger | Behavior |
|--------|---------|----------|
| Add bookmark | Enter URL in add form, click Plus or press Enter | Creates bookmark with auto-detected type. Appears in list immediately |
| Delete bookmark | Click X button (visible on hover) | Removes immediately. No confirmation (consistent with ExternalLinks pattern) |
| Click bookmark | Click bookmark label/URL | Opens in real browser via `openURL` API |

## Data Flow Summary

```
WikiView
├── WikiSidebar
│   ├── Search Input → useDebounceFn(200ms) → wikiStore.doSearch()
│   ├── Page List → wikiStore.sortedPages (computed, A-Z)
│   ├── New Page → wikiStore.createPage()
│   └── Browse ADO Wikis → opens ADOWikiBrowser dialog
├── WikiPageHeader
│   ├── Title → wikiStore.selectedPage.title
│   ├── Tabs → wikiStore.editMode (boolean toggle)
│   └── Delete → wikiStore.deletePage() (after confirmation)
├── WikiEditor (v-if="editMode")
│   └── textarea → wikiStore.updatePage() (debounced auto-save)
└── WikiPreview (v-else)
    └── useMarkdown.render() → sanitized HTML with wiki link handlers

ADOWikiBrowser (Dialog)
├── Wiki Selector → adoWikiStore.fetchWikis()
├── Page Tree → adoWikiStore.fetchPageTree(wikiId)
├── Selection → local ref Set<string>
└── Import → adoWikiStore.importPages() → backend goroutine → events

ProjectBookmarks (in ProjectDetail)
├── Bookmark List → projectStore.bookmarks (per project)
├── Add Form → projectStore.addBookmark()
└── Delete → projectStore.deleteBookmark()
```
