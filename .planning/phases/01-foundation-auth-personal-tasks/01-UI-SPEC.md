# Phase 01 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation, Auth & Personal Tasks.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Design language:** Professional, information-dense, clean — Linear/Notion aesthetic.
> Native desktop feel, not a web app. Engineer-first productivity tool.

### Font Bundle Note

The current `frontend/public/Inter-Medium.ttf` only provides weight 500. Replace with exactly 2 font files:
- Inter-Regular.ttf (weight 400) — body text, descriptions, timestamps, inputs
- Inter-SemiBold.ttf (weight 600) — headings, labels, buttons, badges, stat values

Remove `Inter-Medium.ttf` (weight 500 is no longer used). Alternatively, use `@fontsource-variable/inter` for variable font support. Add `@font-face` declarations in `frontend/public/style.css`.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, table cell padding, tag chip gaps |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, sidebar icon group gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Dashboard section breaks, page top margin below titlebar |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- **macOS titlebar region:** 50px invisible titlebar height (set in `main.go` `InvisibleTitleBarHeight: 50`). The sidebar and content area must account for this — top 50px is drag region.
- **Touch/click targets:** Minimum 32px height for all interactive elements (buttons, sidebar icons, table rows). Sidebar icons use 40×40px hit area.
- **Sidebar width:** 56px collapsed (icon-only). Not on the 8-point scale — this is a layout constant matching Linear/Notion.
- **12px padding:** Used in buttons (horizontal) and table row cells (horizontal). Justified as the midpoint between 8px (too tight for button text) and 16px (too loose for compact controls). Acceptable for inline padding on small interactive elements only.

### Accent Reserved For (explicit list)

The accent color (blue) is reserved ONLY for:
1. Primary CTA buttons ("Create Task", "Sign In")
2. Active/selected sidebar navigation icon background
3. Focus rings on interactive elements (2px ring, offset 2px)
4. Text links in descriptions
5. Command palette selected item highlight
6. Active filter/tab indicator (2px bottom border)

Accent is NOT used for: general text, borders, backgrounds, icons at rest, section headers.

## Interaction Patterns

### Keyboard Shortcuts

| Shortcut | Action | Scope |
|----------|--------|-------|
| `⌘K` / `Ctrl+K` | Open command palette | Global |
| `Esc` | Close command palette / close detail panel / cancel edit | Global |
| `⌘N` / `Ctrl+N` | Create new task | Global (when authenticated) |
| `↑` / `↓` | Navigate task list / command palette results | Task list, command palette |
| `Enter` | Open selected task / execute command | Task list, command palette |
| `⌘Backspace` / `Ctrl+Backspace` | Delete task (with confirmation) | Task detail panel |

Implementation: `@vueuse/core useMagicKeys` for global shortcuts. Headless UI handles focus-trapped keyboard nav in dialogs/comboboxes.

### Transitions

| Element | Enter | Exit |
|---------|-------|------|
| Command palette backdrop | Fade in 150ms ease-out | Fade out 100ms ease-in |
| Command palette panel | Scale 95%→100% + fade in, 150ms ease-out | Scale 100%→95% + fade out, 100ms ease-in |
| Detail side panel | Slide right→left (translateX 100%→0), 200ms ease-out | Slide left→right (0→100%), 150ms ease-in |
| Page transitions | None (instant swap — desktop app, not web nav) |
| Status/priority dropdown | Fade in + translateY -4px→0, 100ms ease-out | Fade out, 75ms ease-in |
| Toast notifications | Slide in from top-right, 200ms ease-out | Fade out, 150ms ease-in |
| Sidebar tooltip | Fade in after 500ms hover delay, 100ms | Fade out 75ms |

Implementation: `@headlessui/vue` `TransitionRoot` + `TransitionChild` for dialog-based transitions. CSS transitions for simple hover/show effects.

### Loading States

| State | Visual |
|-------|--------|
| App launch (restoring session) | Full-screen centered: app icon + "Signing in…" + `Loader2` spin animation (zinc-400, 24×24px) |
| Sign-in in progress | Button shows spinner + "Signing in…", disabled state |
| Task list loading | 5 skeleton rows: 44px height each, zinc-800/50 (dark) / zinc-200/50 (light) pulsing bg, rounded-md |
| Task save | Optimistic update (instant UI change). If backend errors, revert + show toast error. |
| Command palette search | No spinner — results filter as you type (all data is local SQLite, <10ms) |

### Error Handling (UI)

| Error | Display |
|-------|---------|
| Auth failure | Login view: red error text below buttons. Toast in app: "Session expired — sign in again." |
| Task operation fail | Toast notification top-right: red-tinted bg, error message, 5s auto-dismiss. Revert optimistic UI change. |
| Circular dependency | Inline error below dependency add field: "Can't add — this would create a circular dependency: Task A → Task B → Task A" in red-400 (dark) / red-600 (light), 12px |
| Network (Graph API) | Toast: "Couldn't load profile photo. Using initials instead." — non-blocking, 3s dismiss |

## Theme Switching Implementation

Source: D-04 — follow system preference

```typescript
// frontend/src/composables/useTheme.ts
import { useColorMode } from '@vueuse/core'

// Automatically adds/removes 'dark' class on <html> element
// Watches system preference changes in real-time
const mode = useColorMode({
  attribute: 'class',
  modes: { dark: 'dark', light: '' }
})
```

The `dark:` variant in Tailwind CSS v4 responds to the `.dark` class. All color declarations in this contract use the `dark:` pattern:

```html
<div class="bg-white dark:bg-zinc-950 text-zinc-900 dark:text-zinc-100">
```
