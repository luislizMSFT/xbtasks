# Phase 04 — UI Design Contract

## Interaction Patterns

### Graph Node Selection Flow

1. User clicks node in Relationships graph
2. Node receives blue-500 2px ring
3. All edges connected to node (direct + transitive chain) highlight to blue-400
4. All connected nodes get subtle highlight (border opacity increase to 80%)
5. Detail panel slides in from right (200ms ease-out)
6. Click same node again or press Escape → deselect, panel slides out
7. Click different node → panel content transitions (no slide out/in, just content swap)

### PR Snooze Flow

1. User hovers PR row → snooze `Clock` icon appears (opacity-0 → opacity-100, 150ms)
2. User clicks snooze icon → DropdownMenu opens
3. User selects duration → PR row becomes snoozed (opacity-60, amber left border)
4. Snooze stored locally: `pr_snoozes` table with `pr_id`, `snooze_until` timestamp
5. On next data refresh: if `snooze_until > now()`, PR is hidden from active sections
6. When snooze expires: PR reappears in active section normally

### PR Merge Nudge Flow

1. Sync detects linked PR status changed to "completed"
2. Toast appears (bottom-right, 10s duration)
3. User clicks "Mark Task Done" → task status set to "done", toast dismisses
4. User clicks "Dismiss" or toast auto-closes → no change, nudge not repeated for same PR

### AB# Detection Flow

1. On PR data load/refresh, parse each PR title + description for `/AB#(\d+)/g` pattern
2. If AB# found and not already linked → show suggestion banner inline on PR row
3. User clicks "Link Work Item" → create task↔ADO link (reuse LinkService)
4. User clicks "Skip" → store dismissal in local state, banner hidden permanently for that PR+ADO pair

### Timeline Interaction

1. Default: show last 30 days of events
2. Horizontal scroll or drag to pan along time axis
3. Ctrl+scroll (or pinch on trackpad) to zoom time scale
4. Click event card → navigate to entity
5. Hover event card → card elevates, shows full title if truncated
